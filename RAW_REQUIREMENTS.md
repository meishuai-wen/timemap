# 项目代号：时空纪元 (TimeMap)
## 项目愿景
做一个交互式的 3D 世界地图网站，展示和查询历史上同一天（或某一年）在全球各地发生的大事。打破单一文明的线性历史观，提供一种“上帝视角”的共时性历史体验。

## 核心功能模块 (MVP)
1. **交互式时空地图**：基于 3D 地球（如 Cesium/Three.js），按地理坐标和时间轴展示历史事件（发光锚点）。
2. **事件沉浸式面板**：点击锚点，展示事件详情（标题、时间、图片、维基百科摘要、所属分类标签）。
3. **因果关联（蝴蝶效应）**：选中某个重大历史事件时，地图上会高亮并连线展示由它引发的后续相关事件。

## 进阶高能功能 (V2.0 规划：平行时空全景)
4. **横向历史对比模式**：
   - **动态疆域叠加**：全球政治版图动态浮现（如大秦、罗马）。
   - **全球文明仪表盘**：东西方帝王、朝代对比。
   - **同代人物碰撞**：展示同时代活跃的思想家、科学家。

## 深度硬核功能 (V3.0 规划：文明透视仪 Civilization X-Ray)
5. **大国多维数据看板 (Empire Stats Dashboard)**：
   - **军力与武库 (Military & Armory)**：
     - **军队规模**：常备军数量、兵种构成（步兵/骑兵/战车）。
     - **装备图鉴**：武器与护甲的 3D/2D 样式展示、材料解析（青铜 vs 铁器）、制造规制。
   - **民生与经济 (Socio-Economics)**：
     - **人口与生存**：国家总人口预估、平均寿命、阶级结构。
     - **生活质量**：平民阶层的主食结构（粟米 vs 小麦）、流通货币、赋税劳役压力。

## 数据架构与持续更新机制 (Data Ops & Updatability)
6. **历史数据的生长模型 (History as Code)**：
   - **AI 全自动清洗流水线**：后端部署定期任务（Cron），监控维基百科更新或全球最新考古发现，利用大模型自动将非结构化新闻清洗为标准的“经纬度+年份+标签”的 JSON 数据包入库。
   - **Wiki 式众包纠错与共建 (UGC)**：硬核历史产品必有争议点（如某战役实际伤亡人数）。前端提供“提出异议/报错”按钮，允许历史系学生或专家提交史料来源。
   - **版本化发布 (DLC 模式)**：由于无法一次性覆盖全人类历史，数据采用“资料包（DLC）”的形式迭代。例如：首发涵盖“轴心时代”，下个月更新“v1.2 蒙古西征资料包”或“v1.3 大航海时代”，通过 Git 的方式管理 JSON 数据文件。
   - **Agent 自动化审核初审**：当用户提交纠错时，引入专门的“历史考证 Agent”，自动去多语言维基库交叉比对，出具可信度评分后交由人类管理员“一键 Merge”。
