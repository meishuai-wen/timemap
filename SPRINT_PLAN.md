# 《时空纪元 (TimeMap)》下半场冲刺计划与排期

## 核心目标
全面推进并落地「时空纪元」剩余核心需求，完成从数据采集到前端高阶交互的闭环，实现 V2.0 体验跃升及 V3.0 高阶功能点亮。

## 阶段目标与开发顺序

### 第一阶段：数据中台建设（基石阶段）
**目标**：构建稳定、高质量的历史数据源，为前端动效和面板提供数据支撑。
*   **核心任务**：Python 爬虫 + LLM 数据清洗流水线搭建。
*   **关键交付物**：
    1. 自动化历史事件与边界数据抓取脚本（Python）。
    2. 集成 LLM 的结构化数据清洗与标签化服务。
    3. 核心 API 接口输出（供前端调用）。
*   **排期估算**：第 1 周

### 第二阶段：前端体验收尾 (V2.0 升级)
**目标**：强化视觉冲击力与交互深度，完成 V2.0 版本的核心视觉表现。
*   **核心任务**：2D 蝴蝶效应连线、动态疆域晕染。
*   **关键交付物**：
    1. 历史事件因果链条的 2D 视觉连线（蝴蝶效应动画）。
    2. 基于时间轴的帝国/政权疆域动态晕染与扩张演变效果。
*   **排期估算**：第 2 周

### 第三阶段：高阶功能突破 (V3.0 预演)
**目标**：拓展应用深度，引入多维度的历史数据分析面板。
*   **核心任务**：文明透视仪 (V3.0) 军力民生面板开发。
*   **关键交付物**：
    1. 文明透视仪 UI/UX 框架搭建。
    2. 军力指数与民生指数可视化多维面板。
    3. 数据中台与高阶面板的数据对接联调。
*   **排期估算**：第 3 周 - 第 4 周

## 风险管理
*   **数据准确性**：LLM 清洗历史数据可能存在幻觉，需增加置信度评估或抽样人工复核。
*   **前端性能**：动态疆域晕染和大规模连线可能导致掉帧，需提前进行渲染性能（WebGL/Canvas）压测与优化。
