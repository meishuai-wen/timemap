# 产品需求文档 (PRD) - 时空纪元 (TimeMap)

## 1. 项目背景与愿景
当前市面上绝大多数历史科普产品与教学工具均采用“单一文明、线性时间”的叙事结构，用户难以直观感知同一历史截面下全球各地的文明碰撞与共时性关联。
**“时空纪元 (TimeMap)”** 旨在打造一个交互式的 3D 世界地图产品。本项目打破传统的线性历史观，为用户提供一种“上帝视角”的共时性历史体验，直观展示和查询历史上同一天（或某一年）在全球各地发生的大事与文明变迁。

## 2. 目标受众 (Target Audience)
1. **历史爱好者与极客**：对历史横向对比、冷门史料挖掘有浓厚兴趣的硬核用户。
2. **学生与教育工作者**：需要构建全球化历史大局观、直观演示历史事件地理与时间维度的师生群体。
3. **策略游戏玩家群体**：习惯于P社（Paradox）、《文明》系列等大战略游戏，对地缘政治、军事与民生数据面板有极高偏好的玩家。
4. **泛科普大众用户**：对“蝴蝶效应”、历史冷知识感兴趣，易被高颜值 3D 沉浸式交互吸引的普通网民。

## 3. 核心功能模块规划
产品将采用敏捷迭代策略，按 MVP、V2.0、V3.0 三个大阶段进行产品交付。

### Phase 1: MVP（最小可行性产品）- 基础共时性地图
*   **交互式时空地图**：基于 3D 地球引擎（如 Cesium / Three.js）构建底图。支持时间轴（Timeline）拖拽，按所选时间点及地理坐标，在地图上以“发光锚点”形式呈现历史事件。
*   **事件沉浸式面板**：用户点击地图锚点后，滑出详细信息面板。内容包括：事件标题、精准时间、代表性图片、维基百科摘要（Wiki Summary）以及所属的分类标签（如：政治、军事、文化、科技）。
*   **因果关联（蝴蝶效应）系统**：选中某一重大历史节点事件时，地图动态高亮并以流光连线展示由该事件直接或间接引发的后续事件，具象化历史的因果链条。

### Phase 2: V2.0（进阶高能）- 平行时空全景
*   **横向历史对比模式**：
    *   **动态疆域叠加**：根据时间轴滑动，全球各大帝国/政权的政治版图（如大秦与罗马）在 3D 地球上动态浮现与消亡。
    *   **全球文明仪表盘**：提供宏观视角的对比面板，支持同时期东西方帝王、朝代的国力/治世水平横向对比。
    *   **同代人物碰撞**：通过特定筛选图层，展示同一时期在全球各地活跃的思想家、科学家、将领（例如：展示孔子与释迦牟尼、毕达哥拉斯的共时性）。

### Phase 3: V3.0（深度硬核）- 文明透视仪 (Civilization X-Ray)
*   **大国多维数据看板 (Empire Stats Dashboard)**：
    *   **军力与武库 (Military & Armory)**：
        *   **军队基本盘**：常备军规模估算、兵种构成比例（步兵/骑兵/战车等）。
        *   **装备图鉴**：提供武器与护甲的 3D/2D 拆解展示，包含材料演进解析（如青铜过渡到铁器）与制造规制考究。
    *   **民生与经济 (Socio-Economics)**：
        *   **人口与生存状态**：展示各阶段国家总人口预估模型、人均寿命、社会阶级结构比例。
        *   **生活质量指数**：平民阶层的主食结构分布（粟米 vs 小麦）、流通货币体系、赋税与劳役压力可视化。

## 4. 交互与信息架构 (Interaction & Information Architecture)
*   **全局导航与操控**：
    *   **空间维度**：鼠标/手势控制 3D 地球的缩放（Zoom）、平移（Pan）、旋转（Rotate）。
    *   **时间维度**：底部常驻全局时间轴滑块（Timeline Slider），支持按年/世纪快速跳转，或以特定速率自动播放历史演进。
*   **图层与视图切换 (Layers & Views)**：
    *   默认视图：事件锚点层。
    *   可选图层叠加：疆域板块层、人口热力层、商贸路线层（如丝绸之路）。
*   **层级下钻 (Drill-down)**：
    *   `L1 宏观层`：旋转地球，观看高亮节点与疆域色块。
    *   `L2 详情层`：点击节点，右侧或左侧弹出“事件沉浸式面板”或“文明透视仪”。
    *   `L3 关联层`：在详情面板点击“蝴蝶效应”按钮，视角自动缩放并拉拽连线至相关历史事件。

## 5. 数据架构与持续更新机制 (Data Ops & Updatability)
基于“History as Code（历史即代码）”理念，构建可持续生长的底层数据生态：
1.  **AI 全自动清洗流水线**：
    *   后端部署 Cron 定时任务，实时监控维基百科更新与全球权威考古新闻发布。
    *   接入大模型（LLM）构建清洗 Agent，自动将非结构化文本抽取并清洗为标准化的 JSON 数据包（包含字段：`经纬度`、`年份/日期`、`事件摘要`、`分类标签`），自动入库。
2.  **Wiki 式众包纠错与共建 (UGC 体系)**：
    *   针对历史学界存在的争议点（如某场战役的具体伤亡规模），前端面向高级用户提供“提出异议/报错”入口。
    *   鼓励历史系学生、专家学者提交具有史料来源的修正建议，形成高智力密度的社区氛围。
3.  **Agent 自动化初审机制**：
    *   当用户提交 UGC 纠错时，系统自动唤醒“历史考证 Agent”。
    *   Agent 会自动检索多语言维基百科、学术数据库进行交叉比对，出具包含史料支撑的“可信度评分”报告，并最终交由人类管理员执行“一键 Merge”。
4.  **版本化发布策略 (DLC 模式)**：
    *   鉴于全人类历史数据过于浩瀚，项目采取 Git 化的 JSON 数据文件管理方式，以“资料包（DLC）”形态进行敏捷发版。
    *   首发版本覆盖核心节点（如“轴心时代”基础包），后续每月按主题推送数据更新（如“v1.2 蒙古西征资料包”、“v1.3 大航海时代资料包”），确保内容的持续新鲜感与交付质量。
